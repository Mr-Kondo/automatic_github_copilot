- ユーザー登録API

- メールアドレス形式が不正な場合は 400 系エラーで拒否すること。

- pytest のユニットテストを追加または更新すること。
- 既存の API インターフェース変更は必要最小限にとどめること。
- 無関係なファイル変更や大規模リファクタリングは行わないこと。

レビュー観点:
- バリデーション漏れがないか
- 異常系テストが十分か
- エラーハンドリングが過不足ないか
- 変更が局所的にとどまっているか

- 変更したファイル
- 実装内容
- 実行したテスト
- 残る懸念点
